# Security Policy

## Supported Versions

VietSalePro is currently under active development.

Security updates are provided only for the latest development version maintained by the project owner.

Older versions are not guaranteed to receive security fixes.

---

## Reporting a Vulnerability

If you discover a security vulnerability in VietSalePro, please report it responsibly.

Please include as much information as possible:

* Description of the vulnerability
* Steps to reproduce
* Expected behavior
* Actual behavior
* Screenshots or proof of concept (if applicable)
* Suggested mitigation (optional)

Please **do not publicly disclose** security vulnerabilities before they have been reviewed and resolved.

---

## Responsible Disclosure

To help protect users and the integrity of the project, we request that security issues be reported privately to the project owner.

Public disclosure before a fix is available may place users at unnecessary risk.

---

## Scope

This policy applies to all components of the VietSalePro project, including but not limited to:

* Source Code
* Desktop Application
* Mobile Application
* Web Application
* API Services
* Database
* Authentication
* Authorization
* Supabase Integration
* Deployment Infrastructure
* Documentation

---

## Security Best Practices

The project follows modern software engineering practices, including:

* Secure authentication and authorization
* Principle of least privilege
* Regular dependency updates
* Input validation
* Secure API design
* Environment variable protection
* Secrets excluded from version control
* Continuous code review
* AI-assisted development with human verification

---

## Third-Party Components

VietSalePro uses third-party libraries and services.

Security issues affecting third-party dependencies should be reported according to the respective vendor's security policy whenever appropriate.

---

## Response Process

Reported security issues will generally follow this process:

1. Vulnerability report received.
2. Initial assessment and validation.
3. Severity classification.
4. Investigation and remediation.
5. Security testing.
6. Deployment of the fix.
7. Notification to affected users (when applicable).

Response times may vary depending on the complexity and severity of the issue.

---

## Disclaimer

This project is proprietary software.

Submission of a vulnerability report does not grant any ownership, license, or rights to the reporter.

The project owner reserves the right to determine the appropriate remediation process and disclosure timeline.

---

Thank you for helping improve the security of VietSalePro.
